# chrome-stock-monitor

一个可直接加载到 Chrome 的 Manifest V3 股票监控插件，专门为 popup 场景做了轻量化和紧凑化处理。当前版本继续保持无后端依赖，支持 A 股、港股、美股三栏查看，手动添加股票、手动/自动刷新、本地持久化和 badge 提醒。

## 已实现功能

- 紧凑 popup UI：
  - 顶部工具区改为更轻的状态条
  - 首屏重点保留三栏行情、搜索与关键操作
  - 刷新 / 导入示例 / 设置改为图标化按钮，并保留可用性提示
  - 三个市场列表支持独立滚动，首屏信息密度更高
- 股票搜索：
  - A 股 / 港股升级为远程全市场检索，优先走腾讯 `smartbox` 搜索并写入本地缓存
  - 支持按代码、名称、拼音、缩写搜索，常用查询命中后可直接走本地缓存
  - 美股继续使用内置热门股票库，支持代码、名称、别名匹配
  - 搜索结果优先展示远程检索命中的 A 股 / 港股股票，回退到本地热门库兜底
  - 继续支持按代码直接添加：
    - A 股：`600519`、`830799`（北交所也支持）
    - 港股：`00700`
    - 美股：`AAPL`
- 行情刷新：
  - popup 内手动刷新
  - 后台自动刷新（`chrome.alarms`）
- 行情展示：
  - 最新价格
  - 涨跌额
  - 涨跌幅
  - 更新时间
  - 红涨绿跌
- 提醒：
  - 单只股票可设置涨跌幅阈值
  - 单只股票可设置目标价和方向（`>=` / `<=`）
  - 设置面板可控制 badge、涨跌幅提醒、目标价提醒
  - badge 展示当前活跃提醒数量
- 本地持久化：
  - 观察列表
  - 全局设置
  - 单股提醒阈值
  - 提醒触发状态
  - 最近一次刷新状态
- 状态处理：
  - 空状态
  - 加载状态
  - 错误状态
- 附带示例股票池，一键导入便于体验

## 搜索覆盖

当前搜索能力拆成两层：

- A 股 / 港股：优先调用腾讯 `smartbox` 搜索接口做全市场候选检索，并把查询结果写入 `chrome.storage.local` 缓存
- 美股：继续使用内置热门股票库，覆盖常见高频标的
- 本地热门库：继续作为 A 股 / 港股搜索失败时的兜底，也用于示例股票池

说明：

- A 股 / 港股的名称、代码、拼音、简称搜索覆盖已从“热门内置库”升级为“尽可能走全市场远程检索”
- 同一查询命中缓存后，后续搜索不会重复打远程接口，popup 体验更稳
- 代码直加仍然可用，只要行情源支持且代码格式正确，就可以添加
- A 股代码直加已补上北交所前缀推断（如 `830799` → `bj830799`）

## 加载方式

1. 打开 Chrome，进入 `chrome://extensions/`
2. 打开右上角“开发者模式”
3. 选择“加载已解压的扩展程序”
4. 选择当前目录 `/Users/yhl/develop/code/chrome-stock-monitor`
5. 固定插件到工具栏后，点击图标即可打开 popup

## 数据来源

- 行情接口：`https://qt.gtimg.cn/`
- A 股 / 港股搜索接口：`https://smartbox.gtimg.cn/`
- 本地搜索数据：插件内置的热门股票搜索库（用于美股和 A/H 兜底）

## 示例股票池

- A 股：贵州茅台、宁德时代、比亚迪
- 港股：腾讯控股、小米集团、美团
- 美股：Apple、NVIDIA、Tesla

## 限制项

- 当前使用公开行情 / 搜索接口，稳定性和字段格式不受插件控制
- 当前没有交易日历和交易时段判断，非交易时段可能显示上次成交数据
- A 股 / 港股虽然已切到尽可能全量的远程检索，但结果质量仍受上游搜索接口影响
- 美股暂时仍是本地热门库，不是完整全市场
- 当前提醒只实现 badge，不包含桌面通知、声音、消息推送
- 当前没有排序、分组、拖拽、自定义主题等扩展功能

## 开发说明

- 纯原生 HTML / CSS / JavaScript
- 不依赖后端服务
- 不引入重量级前端框架
- 所有持久化都写入 `chrome.storage.local`
